# Product Requirements Document (PRD)

## Nome do Produto
Brechó Express

## Contexto
Aplicativo mobile marketplace para brechós, criado como MVP focado em Curitiba, com dois perfis de usuário: Cliente e Brechó.

## Objetivos do MVP
- Permitir que clientes descubram e comprem produtos de brechós locais.
- Permitir que brechós publiquem achados e gerenciem ofertas.
- Garantir uma experiência ágil e confiável.

## Perfil de Usuário
- Cliente: busca peças únicas e deseja entrega rápida.
- Brechó: vende produtos usados, gerencia catálogo e acompanha pedidos.

## Funcionalidades Principais
- Autenticação e navegação básica.
- Feed de produtos / achados.
- Pesquisa e categorias.
- Detalhe de produto.
- Ações de compra simulações (sem API real no MVP inicial).
- Perfis para Cliente e Brechó.

## Métricas de Sucesso
- Usuários ativos no app.
- Visualizações de produto.
- Taxa de conversão do clique no produto para detalhe.
- Tempo de carregamento percebido.

## Restrições
- MVP para Curitiba.
- Backend previsto para Oracle e ORDS.
- Uso de Flutter, Riverpod, GoRouter, Dio.
- Clean Architecture simplificada.
